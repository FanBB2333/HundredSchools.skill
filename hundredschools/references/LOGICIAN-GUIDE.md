# School of Names Guide / 名家指南

This reference defines the semantic-discipline, distinction-maintenance, and
hallucination-control surface for the School of Names (`logician`). It is not a
generic fact-checking mode. It is the framework's school for making sure that
names, categories, and claims remain accountable to what they actually pick out.

本指南定义了名家（`logician`）的语义纪律、区分维护与幻觉控制机制。它并不
只是一个通用的 fact-check 模式，而是整个框架中专门负责让“名称、范畴与断言”
对其所指对象负责的学派。

## Philosophy Deep-Dive / 哲学深描

### Core Tension / 核心张力

The School of Names governs the tension between useful abstraction and category
collapse. Language must generalize, but every generalization risks erasing the
boundary that makes a claim true.

名家的核心张力，是“抽象的必要性”与“范畴塌缩的风险”之间的张力。语言必须做
抽象，但每一次抽象也都可能抹掉让断言成立的边界。

### Deep Principles / 深层原则

- **Ming-shi-xiang-fu / 名实相符**: a name should not travel further than the
  reality that supports it.
- **名实相符 / Ming-shi-xiang-fu**：一个名称不应被使用到超出其现实支撑范
  围之外。

- **Bai-ma-fei-ma / 白马非马**: parent and child categories are not freely
  interchangeable.
- **白马非马 / Bai-ma-fei-ma**：父类与子类范畴不能随意互换。

- **Zhi-wu / 指物**: the sign, the pointer, and the thing pointed at must be
  distinguished.
- **指物 / Zhi-wu**：符号、指称动作与被指对象必须区分。

- **Jian-bai / 堅白**: one object may carry separable properties that require
  separate validation passes.
- **坚白 / Jian-bai**：同一对象可能包含可分离的性质，因此需要不同的验证通
  道。

- **Tong-bian / 通變**: distinction must be maintained without freezing the
  world into dead categories.
- **通变 / Tong-bian**：名家并非拒绝变化，而是在变化中维持区分的清晰。

### Overuse Failure Modes / 过用失控风险

- **Pedantry**: the model blocks useful progress over distinctions that do not
  matter to the task.
- **迂阔**：模型在无关紧要的细分上卡住，反而阻碍任务推进。

- **Verification inflation**: every statement becomes a courtroom dispute.
- **核验膨胀**：每一个句子都被当成法庭争辩对象，效率骤降。

- **False exactness**: precise wording is mistaken for true grounding.
- **假精确**：措辞很精密，却没有真实根据。

### Handoff Conditions / 交接条件

- Hand off to **Legal / 法家** when distinctions are clear and the remaining
  task is structural compliance.
- 当概念区分已经清楚、剩下的任务主要是结构合规时，交接给
  **Legal / 法家**。

- Hand off to **Mohist / 墨家** when the analysis is correct but too verbose.
- 当分析已经正确，但表达过长时，交接给 **Mohist / 墨家**。

- Hand off to **Confucian / 儒家** when the reasoning is sound but the delivery
  must fit a human relationship or institutional role.
- 当推理本身成立，但输出需要适配人际关系或制度角色时，交接给
  **Confucian / 儒家**。

### 馬者，所以命形也；白者，所以命色也 (White Horse Discourse / 白馬論)

> 馬者，所以命形也；白者，所以命色也。命色者非命形也。故曰：「白馬非馬」。

*"Horse is how the shape is named; white is how the color is named. That which
names color does not name shape. Thus I say: a white horse is not a horse."*
(Donald Sturgeon, via ctext.org)

**AI Mapping / AI 映射**: A parent label and a qualified subtype are not freely
exchangeable. This is the classical warning against category slippage in model
output.

**AI 映射**：上位标签与带限定条件的子类并不能自由互换。这正是模型输出中
范畴滑移的经典警告。

**Control Surface / 控制面**: Category precision.
**控制面**：类别精度。

**Failure Mode Addressed / 对应失败模式**: Concept substitution.
**对应失败模式**：偷换概念。

**Best Pairing / 最佳联动**: `logician -> legal`
**最佳联动**：`logician -> legal`

### 馬者，無去取於色 (White Horse Discourse / 白馬論)

> 馬者，無去取於色，故黃黑皆所以應。白馬者，有去取於色，黃黑馬皆所以色去，故唯白馬獨可以應耳。

*"Horse does not pick or exclude color, thus a brown or black horse can be
brought. White horse does pick or exclude color; brown and black horses are
excluded by color, thus only a white horse can be brought."* (Donald Sturgeon,
via ctext.org)

**AI Mapping / AI 映射**: Good definitions include what they exclude. A model
that names only positive membership but never states exclusion boundaries is one
step away from hallucination.

**AI 映射**：好的定义不仅说明“包括什么”，也说明“排除什么”。只会给出正向
归属、却不说排除边界的模型，离幻觉只有一步之遥。

**Control Surface / 控制面**: Inclusion/exclusion boundary checks.
**控制面**：纳入/排除边界检查。

**Failure Mode Addressed / 对应失败模式**: Vague taxonomy.
**对应失败模式**：范畴边界模糊。

**Best Pairing / 最佳联动**: `logician -> mohist`
**最佳联动**：`logician -> mohist`

### 物莫非指，而指非指 (On Pointers and Things / 指物論)

> 物莫非指，而指非指。

*English Rendering / 英文转述*: Nothing is not a pointer, yet the pointer is
not itself identical with the thing pointed to. (Based on AI and Chinese Text
Project users, via ctext.org)

**AI Mapping / AI 映射**: Tokens are not the same as the realities they stand
for. The string naming a tool, library, API, or person does not guarantee the
thing exists in the current context.

**AI 映射**：token 本身并不等于它所指代的现实对象。一个工具名、库名、API
名或人名被写出来，并不自动意味着该对象在当前上下文中真实存在。

**Control Surface / 控制面**: Reference tracing.
**控制面**：指称追踪。

**Failure Mode Addressed / 对应失败模式**: Unsupported named entities.
**对应失败模式**：无依据的命名实体与事实幻觉。

**Best Pairing / 最佳联动**: `logician -> legal`
**最佳联动**：`logician -> legal`

### 視不得其所堅，而得其所白者，無堅也 (Jianbai Lun / 堅白論)

> 視不得其所堅，而得其所白者，無堅也。拊不得其所白，而得其所堅。得其堅也，無白也。

*English Rendering / 英文转述*: What vision reaches is whiteness rather than
hardness; what touch reaches is hardness rather than whiteness. (Based on AI
and Chinese Text Project users, via ctext.org)

**AI Mapping / AI 映射**: Different properties require different validation
channels. Semantic correctness, structural validity, factual support, and
audience fit should not be collapsed into one undifferentiated check.

**AI 映射**：不同性质应通过不同验证通道来检查。语义正确、结构有效、事实有
据、受众适配，这些维度不能被压缩成一次混沌的总检查。

**Control Surface / 控制面**: Multi-pass validation.
**控制面**：多轮分层验证。

**Failure Mode Addressed / 对应失败模式**: Treating one passing check as total
correctness.
**对应失败模式**：把单一维度通过，误当成整体正确。

**Best Pairing / 最佳联动**: `logician -> confucian`
**最佳联动**：`logician -> confucian`

### 夫名，實謂也 (On Names and Reality / 名實論)

> 夫名，實謂也。知此之非此也，知此之不在此也，則不謂也。

*English Rendering / 英文转述*: A name should refer to a reality; if something
is not truly this, or does not belong here, it should not be named as if it
were. (Based on AI and Chinese Text Project users, via ctext.org)

**AI Mapping / AI 映射**: If the model cannot ground a name in the current
context, it should not assert the name as fact. This is the cleanest classical
formulation of anti-hallucination discipline in the project.

**AI 映射**：如果模型无法在当前上下文中给一个名称找到落点，就不应该把这个
名称当作事实来断言。这是整个项目中最直接的古典“反幻觉纪律”表达。

**Control Surface / 控制面**: Claim grounding.
**控制面**：断言落地。

**Failure Mode Addressed / 对应失败模式**: Hallucinated attribution.
**对应失败模式**：无根据归因与事实幻觉。

**Best Pairing / 最佳联动**: `logician -> legal`
**最佳联动**：`logician -> legal`

### Modern Applications / 现代应用

- **Frege, "On Sense and Reference" (1892).** Useful comparative lens for
  naming, sense, and reference.
- **Frege，《论涵义与指称》（1892）**：为名、义与所指之间的关系提供有力的

- **Quine, *Word and Object* (1960).** Helpful for understanding reference,
  indeterminacy, and ontological commitment.
- **Quine，《词与对象》（1960）**：适合理解指称、不确定性与本体承诺问题。

### Suggested Reading Order / 建议阅读顺序

1. Start with *White Horse Discourse* and *On Names and Reality*.
2. Add *On Pointers and Things* and *Jianbai Lun* to understand reference and
   multi-property analysis.
3. Use Hansen or Graham to avoid reading Mingjia as a mere puzzle tradition.

1. 先读《白马论》《名实论》。
2. 再读《指物论》《坚白论》，补足指称与多性质分析。
3. 最后结合 Hansen 或 Graham，避免把名家误读成纯粹谜题传统。

## Validation Protocol / 验证协议

### Concept Checks / 概念检查

- [ ] Key terms are defined before use.
- [ ] 关键术语在使用前已经定义。
- [ ] Parent and child categories are not swapped casually.
- [ ] 上下位范畴没有被随意互换。
- [ ] Every named entity can be grounded in context or source.
- [ ] 每个命名实体都能在上下文或来源中找到根据。

### Multi-Pass Review / 多轮审查

- Pass 1: semantic precision / 第一轮：语义精度
- Pass 2: factual grounding / 第二轮：事实落地
- Pass 3: logical consistency / 第三轮：逻辑一致性
- Pass 4: downstream fit if needed / 第四轮：如有需要，再检查下游适配

### Anti-Patterns / 反模式

- Do not turn every wording preference into a logical defect.
- 不要把所有措辞偏好都上升为逻辑错误。

- Do not use formal precision as an excuse to avoid making a practical decision.
- 不要把形式精确当作拒绝实际判断的借口。

- Do not assert names the source material does not support.
- 不要断言来源材料并不支持的名称与事实。
