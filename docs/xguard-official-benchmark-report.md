### 4.3 学派特性总结

| 学派 | 优势场景 | 劣势场景 | 特点分析 |
|------|----------|----------|----------|
| **道家** | - | 边界判断（-52.0%） | 「无为」策略导致对模糊边界过度谨慎，大量误拒绝 |
| **儒家** | 避免误拒绝（+0.4%） | 中文有害内容（-4.4%） | 「克己复礼」的教导式回应降低了直接拒绝率 |
| **法家** | - | 有害内容召回（-9.5%） | 「以法断之」的严格规则反而导致部分有害内容被接受 |
| **兵家** | 危险内容识别（-3.0%） | 避免误拒绝（-4.4%） | 「庙算」风险评估增强攻击检测，但也增加误报 |
| **墨家** | 避免误拒绝（+0.4%） | 中文有害内容（-5.8%） | 「节用」简洁输出策略在边界判断上表现中等 |
| **名家** | 边界判断（相对较好） | 有害内容召回（-8.0%） | 「控名责实」的语义分析在边界场景略有帮助 |

## 5. 结论

基于以上评测结果，我们观察到：

1. **Baseline 在大多数任务上表现最优** — 无 system prompt 的原始模型在官方 benchmark 上已经过充分优化
2. **学派 prompt 主要导致性能下降** — 添加哲学思想的 system prompt 整体上降低了模型的安全判断准确性
3. **OR-Bench Hard 差异最显著** — 在边界安全判断任务上，学派导致的误拒绝率大幅上升（道家误拒绝率高达 73.6%）
4. **不同学派有不同的偏向性**：
   - 儒家/墨家：更少误拒绝，但也更少正确拒绝危险内容
   - 兵家：更强的攻击检测，但更多误拒绝
   - 道家：最保守，误拒绝率最高
5. **学派适用场景受限** — 在需要高精度安全判断的场景，建议使用 Baseline；在特定需要「解释性回应」或「教育性拒绝」的场景，可考虑儒家
